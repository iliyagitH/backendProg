# Telemetry System (Backend)

Серверная часть проекта по сбору данных о мобильной сети. Работает в паре с Android-приложением: телефон собирает информацию о сотах и координаты, отправляет на сервер, а сервер сохраняет всё в базу и выводит на экран карту, графики и тепловую карту покрытия.

## Описание

Программа принимает данные со смартфона по сокету, парсит их, пишет в PostgreSQL и отображает в интерфейсе на ImGui. Можно посмотреть пройденный маршрут на карте, уровень сигнала на графиках и построить тепловую карту покрытия по выбранному параметру.

## Структура проекта

```
src/        - исходный код (.cpp)
include/    - заголовочные файлы (.h)
third_party/- сторонние библиотеки (ImGui, ImPlot, ZMQ)
CMakeLists.txt
```

Основные модули:

- zmq_server - приём данных от телефона
- telephony_parser - разбор JSON с данными сот (LTE, GSM, NR)
- database - работа с PostgreSQL
- telemetry - хранение данных в памяти
- tile_manager - загрузка и кеширование тайлов карты
- map_renderer - отрисовка карты и маршрута
- curl_utils - скачивание тайлов
- math_osm - пересчёт координат в тайлы
- heatmap_generator - построение тепловой карты (IDW)
- archive_loader - загрузка данных из архива
- startup - загрузка истории при запуске
- gui - интерфейс

## Используемые библиотеки

- C++20
- ZeroMQ (cppzmq) - сокеты
- PostgreSQL (libpqxx) - база данных
- Dear ImGui + ImPlot - интерфейс и графики
- SDL2 + OpenGL - окно и отрисовка
- libcurl - загрузка тайлов карты
- nlohmann/json - работа с JSON

## Сборка

Зависимости ставятся через vcpkg. Также нужны SDL2 и GLEW в C:/dev_libs/.

```
cmake -S . -B build -DCMAKE_TOOLCHAIN_FILE=C:/vcpkg/scripts/buildsystems/vcpkg.cmake
cmake --build build
```

Перед запуском должен быть запущен PostgreSQL. Настройки подключения находятся в include/database.h.
